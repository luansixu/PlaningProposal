# 游戏创意（LLM 核心玩法点子库）

> 目标：收录"只有引入大模型才成立/更有趣"的玩家创意体验，并用「玩家做什么 → LLM做什么 → 影响到的游戏状态」描述，便于直接写入策划案的"LLM 核心玩法闭环"章节。

---

## AI 游戏设计原则（Gameplay-First AI Design）

> **本章是整个创意库的底层设计宪法。** 所有 A/B/C/D 类创意的实现，都必须在这些原则的约束下进行。
> 本章来自对 AIvilization、星露谷物语、星界边境等游戏的对比分析，以及"什么是游戏性"的深度讨论。

### 原则 0：先做好玩的游戏，再加 AI

**错误的优先级**：AI 能力 → 围绕 AI 设计游戏 → 希望它好玩
**正确的优先级**：好玩的游戏 → 找到 AI 能增强的精确位置 → 有限度地接入

> AI 不是游戏的目的，是游戏的手段。玩家不会因为"这个游戏用了大模型"而觉得好玩，只会因为"这个游戏让我体验到了别的游戏给不了的东西"而觉得好玩。

### 原则 1：游戏性金字塔——AI 只在上层介入

```
          ▲  惊喜层（AI 在关键时刻制造独特记忆）
         ╱ ╲            偶发、罕见、高价值
        ╱   ╲
       ╱ 增强层 ╲       AI 让已有系统更丰富（NPC 对话、事件变体）
      ╱           ╲      玩家触发或回合间处理，有限频率
     ╱─────────────╲
    ╱   核心玩法层    ╲   纯游戏机制，不依赖 AI，即时反馈
   ╱                   ╲  资源管理/建造/探索/战斗/制作
  ╱   手感/反馈/节奏层   ╲ 确定性、零延迟、操作即反馈
 ╱───────────────────────╲ 点击/拖拽/移动/收集 → 立刻看到结果
```

**铁律**：
- **底层两层绝对不放 AI。** 手感、节奏、即时反馈必须是确定性的、程序化的、零延迟的。
- **上层两层是 AI 的精确位置。** 在玩家已经享受游戏的前提下，AI 在特定时刻制造惊喜。

### 原则 2：85/15 法则——AI 是调味料，不是主菜

```
玩家的游戏时间分配：

███████████████████████████████░░░░░░
|←── 85% 传统游戏体验（即时反馈）──→||← 15% AI 增强 →|
                                         ↑
                                  但这 15% 是别的游戏
                                  给不了你的、只属于
                                  这个游戏的独特记忆
```

- 85% 的时间，玩家在做传统游戏操作（种地/建造/战斗/探索），享受即时反馈
- 15% 的时间，AI 产出了一个前所未见的事件/对话/反应，创造了独属于这一局的记忆
- **那 15% 不是游戏的全部，但它是玩家会记住并传播的部分**

类比：星露谷 95% 在种地钓鱼，但你记住的是那天下雨在海边遇到 Elliot 然后他说了一段让你动心的话。只不过在我们的游戏里，那段话不是预设的——是 AI 基于全部互动历史生成的、只属于你的话。

### 原则 3：四层反馈速度——从星露谷学到的

好玩的游戏必须在四个时间尺度上都给玩家反馈：

| 层级 | 时间尺度 | 体验感受 | 是否涉及 AI |
|------|---------|---------|------------|
| **秒级循环** | < 1 秒 | 操作→即时反馈（挥锄、点击、拖拽） | **绝不** |
| **分钟级循环** | 1-5 分钟 | 微决策的充实感（今天先干什么？） | **绝不** |
| **天/回合循环** | 5-15 分钟 | 成长与进步（收获/升级/解锁） | **可选增强**（回合间后台处理） |
| **周期/结局循环** | 30分钟-数小时 | 长期目标与情感积累 | **AI 核心价值区** |

**AIvilization 的教训**：它砍掉了秒级和分钟级循环（没有即时操作反馈），直接跳到了"观察涌现层"。就像一个只有剧情没有战斗的 RPG——可能很深刻，但不好玩。

### 原则 4：AI 介入的精确位置——白名单思维

**不是"哪里可以加 AI"，而是"哪里不加 AI 就做不到"。**

#### 绝对不用 AI 的地方（底层手感 + 核心机制）

| 系统 | 为什么不用 AI | 用什么替代 |
|------|-------------|-----------|
| 资源管理（数值计算） | 必须即时、精确、0 延迟 | 纯数值规则 |
| 建造/升级/制作 | 玩家要即时看到结果 | 确定性制作系统 |
| 地图探索/移动 | 操作手感必须丝滑 | 传统游戏逻辑 |
| 天气/季节/时间 | 可预期的节奏感 | 规则表/随机种子 |
| 基础 NPC 日程 | 玩家需要知道"某人下午在哪" | 预设时间表 |
| UI 反馈/动画 | 必须即时响应 | 程序化 |
| 核心胜负判定 | 必须公平透明 | 确定性规则 |

#### 有限度使用 AI 的地方（增强层）

| 场景 | AI 做什么 | 频率控制 | 延迟处理 |
|------|----------|---------|---------|
| 玩家主动找 NPC 对话 | 根据关系/记忆/性格生成对话 | 玩家触发时 | 1-2 秒（玩家主动等待，可接受） |
| NPC 间关系事件 | 生成冲突/合作的具体情节 | 每 2-3 天一次 | 回合间后台处理，玩家不感知 |
| NPC 请求/任务 | 基于当前状态生成个性化请求 | 每 3-5 天一次 | 回合间后台处理 |
| 玩家决策的后果 | 解读政策对每个 NPC 的影响 | 玩家颁布时 | 回合间后台处理 |

#### AI 只在"关键时刻"闪光的地方（惊喜层）

| 场景 | AI 做什么 | 频率 | 效果 |
|------|----------|------|------|
| 突发危机 | 生成前所未有的危机及细节 | 每 5-7 天一次 | 打破常规，制造紧张感 |
| NPC 自发行为 | 做出出乎意料但符合性格的事 | 罕见 | 真正的"涌现惊喜" |
| 结局生成 | 生成编年史和每个角色的结局 | 一局一次 | 强烈的叙事满足感 |

### 原则 5：AI 的输出不是文字墙，而是游戏事件

**错误做法**：AI 输出一大段文字 → 丢给玩家读
**正确做法**：AI 输出结构化 JSON → 游戏系统翻译为事件/动画/简短对话/状态变化

玩家看到的应该是：
- "张三搬走了" + 一个离开动画 + 关系面板变化（而不是 AI 写的一段分析为什么搬走）
- "李四送了你一篮水果" + 资源+3 + 好感度提示（而不是 AI 写的李四的内心独白）
- 只有在玩家**主动查看**时，才展示更详细的日志/日记

### 原则 6：不因为有了 AI 就到处加 AI

> **"能用 AI"不等于"该用 AI"。**

每一个 AI 介入点都必须通过以下检验：

1. **不可替代检验**：去掉 AI，这个功能是否完全无法实现？（如果能用随机表/模板替代到 70% 效果，就不该用 AI）
2. **体验净值检验**：AI 带来的惊喜感是否大于等待延迟带来的挫败感？
3. **频率检验**：这个 AI 功能被触发的频率是否合适？太频繁会让惊喜变成噪音。
4. **降级检验**：AI 失败时的兜底方案是否足够好？是否会破坏游戏体验？

### 设计原则速查表

| 编号 | 原则 | 一句话 |
|------|------|--------|
| 0 | Gameplay First | 先做好玩的游戏，再加 AI |
| 1 | 金字塔法则 | AI 只在上层介入，底层绝不碰 |
| 2 | 85/15 法则 | 85% 传统体验，15% AI 惊喜 |
| 3 | 四层反馈 | 秒级/分钟级反馈绝不依赖 AI |
| 4 | 白名单介入 | 不是"哪里加 AI"，是"哪里非 AI 不可" |
| 5 | 事件化输出 | AI 输出是游戏事件，不是文字墙 |
| 6 | 克制原则 | "能用"不等于"该用"，每个介入点都要检验 |

## 分类总览

所有创意按**玩家与 LLM 的关系**分为四大类：

| 分类 | 核心体验 | 关键词 |
|------|---------|--------|
| **A. 语言即制造** | 玩家用语言"创造"新的游戏机制、物件、能力 | 构筑、编程、发明、定义 |
| **B. 语言即博弈** | 玩家用语言"说服/对抗/欺骗"有理解力的对手 | 辩论、欺骗、推理、对抗 |
| **C. 语言即治理** | 玩家用语言"制定规则/塑造他者行为/操控社会" | 立法、契约、宪法、舆论 |
| **D. LLM 作为世界引擎** | LLM 在后台驱动世界运转，玩家不必"打字"也能体验 AI 原生世界 | 涌现、活世界、隐形引擎、异步回合 |

> **A/B/C** 的共性：语言是玩法本身，玩家明确知道自己在和 LLM 交互。AI 占比高，适合短时高密度体验。
> **D** 的独特性：语言是管道而非玩法，LLM 是玩家看不见的世界引擎。最符合"Gameplay First"原则。

> **重要**：无论哪一类，都必须遵守上方的 AI 游戏设计原则。A/B/C 类虽然 AI 占比更高，但仍需要有不依赖 AI 的基础游戏机制支撑（如卡牌战斗系统、证据收集系统、资源管理系统等）。

## A. 语言即制造（Language as Creation）

> 玩家的自然语言输入被 LLM "编译"为可执行的游戏对象/能力/规则，直接扩展策略空间。

### A1. 语言即技能构筑（Natural-language Deckbuild）

玩家用一句话"定义一张牌/技能意图" → LLM 把意图编译成结构化技能（伤害/代价/触发/稀有度）并自动做平衡约束 → 直接生成可用卡牌并进卡组，影响战斗结果

- **LLM 角色**：技能编译器 + 数值平衡裁判
- **不可替代性**：无 LLM 则无法从自然语言生成结构化技能，退化为"选预设牌"

### A2. 魔法咒语编程（Spell Programming）

玩家写"咒语文本"或自然语言流程（如果…那么…） → LLM 转成受限脚本/状态机并做安全沙箱 → 生成新交互能力，直接改变解谜/战斗策略空间

- **LLM 角色**：自然语言→受限脚本的编译器 + 安全沙箱
- **不可替代性**：无 LLM 就无法解析自由文本咒语，退化为"选预设技能"

### A3. "解释世界"生存（World-Interpretation Survival）

玩家描述行动（如"用潮湿苔藓做过滤器"） → LLM 结合世界规则与物理/生态约束生成可行性与结果 → 直接改变资源、水质、疾病、天气适应等生存变量

- **LLM 角色**：世界物理引擎 + 可行性裁判
- **不可替代性**：无 LLM 无法评估开放式行动的合理性，退化为"选预设行动"

### A4. 语言考古/外星语破译

玩家提出假设与翻译规则 → LLM 生成新语料、用一致性测试"反证/证实"玩家规则 → 语言掌握度提升解锁新交互（控制设备、谈判、解谜），并影响剧情推进

- **LLM 角色**：语言学对手 + 语料生成器 + 一致性验证器
- **不可替代性**：无 LLM 无法动态生成符合隐藏规则的新语料，退化为"查字典"

### A5. "一词塑世"Roguelike（Word-Cost Roguelike）

每个房间/层，玩家只能输入一个"咒语/指令"来改变环境或创造工具。LLM 在世界规则约束下解析意图并生成效果。**用过的词会被"消耗"**（词汇资源化），迫使玩家不断用更创意的表达应对新挑战。

- **核心循环**：遭遇挑战 → 输入咒语 → LLM 在规则约束下生成效果 → 环境/状态改变 → 词汇消耗 → 下一层
- **LLM 角色**：咒语解析器 + 效果生成器 + 词汇消耗裁判
- **独特点**：**词汇本身是 Roguelike 的资源**——越常用的词越早被消耗，逼玩家"语言创新"
- **不可替代性**：无 LLM 无法理解开放式咒语并生成合理效果
- **8小时友好度**：高——纯文字界面 + 几何色块即可，状态变量少
- **灵感来源**：融合 A2（咒语编程）+ A3（解释世界），增加词汇资源化机制

## B. 语言即博弈（Language as Confrontation）

> 玩家用语言与 LLM 驱动的对手/裁判进行深度博弈，胜负取决于论证质量而非数值堆叠。

### B1. 法庭式推理/辩论战斗

玩家提交证据+陈词 → LLM 扮演检方/辩方/法官，实时生成反驳点并依据"证据一致性评分"裁决 → 裁决改变嫌疑人状态、线索解锁与结局分支

- **LLM 角色**：法官 + 对方律师（双角色）
- **不可替代性**：无 LLM 则无法评估开放式论证的逻辑强度

### B2. 谎言与识别（Deception Sandbox）

玩家要"说服 NPC/骗过守卫/伪造身份" → LLM 对玩家话术做语义审查、漏洞提取、可信度评估并驱动 NPC 策略反应 → 警戒值、关系值、通行权限实时变化

- **LLM 角色**：语义审查员 + NPC 智能决策引擎
- **不可替代性**：无 LLM 则无法评估自由文本的可信度与漏洞

### B3. "导师型 Boss"反向教学战

玩家口头描述自己的战术计划 → LLM 作为 Boss 实时"读懂并针对性进化"反制（但受公平约束） → Boss 技能树、行为权重当场改变，玩家必须迭代策略

- **LLM 角色**：会"读心"的 Boss AI + 公平约束裁判
- **不可替代性**：无 LLM 则 Boss 无法理解并针对玩家的自然语言战术

### B4. 即兴剧团协作闯关（Party Improv Heist）

多人各说一句"我的行动与意图" → LLM 把多意图合并成一次协同行动并解析冲突/协同加成 → 直接输出"成功/失败/部分成功"与后果，改变警戒、路线、道具消耗

- **LLM 角色**：多意图融合引擎 + 协同/冲突裁判
- **不可替代性**：无 LLM 无法实时解析多人自由文本的协同关系

### B5. "AI 法庭"对抗游戏（AI Court PvP）

一方扮演"检方"、一方扮演"辩方"，双方轮流提交证据和陈词。LLM 扮演法官，实时评估论证强度、证据一致性、逻辑漏洞，给出结构化裁决。案件由 LLM 生成（保证每局不同），证据链有隐藏信息需要玩家挖掘。

- **核心循环**：阅读案卷 → 选择证据+撰写陈词 → LLM 裁判评分+指出漏洞 → 对方反驳 → 终局裁决
- **LLM 角色**：案件生成器 + 公正法官 + 逻辑漏洞检测器
- **独特点**：**失败不是数值不够，而是"你的论证有逻辑漏洞"**
- **不可替代性**：无 LLM 无法动态生成案件、评估论证质量、检测逻辑漏洞
- **灵感来源**：B1（法庭辩论）的多人对抗升级版，增加 LLM 生成案件 + 隐藏证据机制

## C. 语言即治理（Language as Governance）

> 玩家用语言制定规则、塑造他者行为、操控社会系统，LLM 负责将语言"立法"编译为可执行约束。

### C1. 可对话的任务生成器（Quest as a Contract）

玩家和"任务委托人"谈判任务条款（报酬、风险、验收） → LLM 把条款固化为可执行契约（目标、失败条件、可选条款） → 契约成为关卡规则，影响可用道具、敌人配置与结算

- **LLM 角色**：契约编译器 + 谈判对手
- **不可替代性**：无 LLM 无法将自由谈判转化为可执行关卡规则

### C2. 动态社会网络操盘

玩家发布言论/写文章/做公关 → LLM 模拟群体舆论传播、阵营解读、KOL 二次创作 → 舆情指数、阵营态度、城市政策与事件链变化

- **LLM 角色**：舆论模拟引擎 + 群体心智模型
- **不可替代性**：无 LLM 无法模拟语言在群体中的传播与变形

### C3. 可协商的世界规则（Rule Negotiation Roguelike）

每层开局玩家可用自然语言"提出一个规则变更" → LLM 将其转为可执行的规则卡并给出代价/副作用 → 规则直接重写关卡生成与战斗参数

- **LLM 角色**：规则编译器 + 代价/副作用评估器
- **不可替代性**：无 LLM 无法将自然语言规则转为可执行参数

### C4. AI 角色"成长由你定义"

玩家用对话给队友下"行为宪法/原则" → LLM 将原则固化为决策偏好与禁令（结构化） → 队友在战斗/剧情中按宪法行动，影响存活、忠诚、结局

- **LLM 角色**：宪法编译器 + 角色决策引擎
- **不可替代性**：无 LLM 无法将自由文本原则转化为角色行为逻辑

### C5. "语言即立法"文明沙盒（Legislation Sandbox）

玩家不直接操控角色，而是用自然语言"颁布法律/制度"给一个小型 AI 社会。LLM 把法律转成可执行规则，Agent 群体在规则下涌现出行为——玩家观察后果，再修法、废法、立新法。

- **核心循环**：立法 → LLM 编译为规则约束 → Agent 群体行为涌现 → 社会指标变化（犯罪率/幸福度/GDP） → 玩家观察+修法
- **LLM 角色**：法律编译器 + Agent 行为引擎 + 社会模拟器
- **失败条件**：社会崩溃（暴动/饥荒/全员躺平）
- **不可替代性**：无 LLM 无法将自然语言法律编译为可执行规则，也无法驱动 Agent 涌现
- **灵感来源**：C3（可协商规则）+ AIvilization（多 Agent 涌现），从"改一条规则"升级为"治理一个社会"

### C6. "契约地牢"（Contract Dungeon）

每层地牢前，玩家和"地牢守门人"谈判入场条件（我带什么进去、你给什么奖励、如果我死了怎么赔）。LLM 把谈判结果编译成可执行契约，契约直接决定关卡规则。Boss 也会"读懂"你的契约找漏洞来针对你。

- **核心循环**：谈判契约 → LLM 编译规则 → 执行关卡（受契约约束） → Boss 利用契约漏洞 → 结算 → 下一层重新谈判
- **LLM 角色**：契约编译器 + 谈判对手 + Boss 漏洞分析器
- **策略深度**：玩家必须在"要更多好处"和"留更少漏洞"之间平衡
- **不可替代性**：无 LLM 无法理解自由谈判、编译契约规则、分析漏洞
- **灵感来源**：C1（任务契约）+ B3（导师 Boss），增加"Boss 读契约找漏洞"的对抗层

## D. LLM 作为世界引擎（LLM as World Engine）

> **核心理念**：LLM 不再是玩家直接对话的对象，而是在后台驱动世界运转的"隐形引擎"。玩家通过常规游戏操作（点击、选择、拖拽）与世界交互，LLM 在后台将操作转化为世界变化。玩家感受到的不是"我在和 AI 聊天"，而是"这个世界是活的"。
>
> **D 类是最符合"Gameplay First"原则的分类**——游戏的基础层完全不依赖 AI，AI 只在增强层和惊喜层发挥作用。

### D 类的设计哲学

#### 为什么需要这个分类？

A/B/C 三类的隐含假设是"玩家想打字"。但大量玩家不想打字——他们想**玩游戏**。LLM 最强的能力不是"生成文本"，而是**"理解意图 + 在约束下生成合理响应"**。文本只是接口，不是价值。

#### 架构模型对比

```
A/B/C 类（语言是玩法）：
  玩家 ←自然语言对话→ LLM → 游戏状态变化

D 类（LLM 是引擎）：
  玩家 ←常规游戏操作→ 游戏系统 ←JSON/结构化数据→ LLM → 游戏系统 → 世界变化
                                    ↑                          ↓
                              翻译层（操作→上下文）      翻译层（决策→表现）
```

玩家可能完全不知道背后有 LLM。他看到的是：
- 一个 NPC 突然做了一个出乎意料但合理的决策
- 两个他养的角色打了一场他预料不到的架
- 一个村庄因为他之前的行为产生了连锁反应
- 世界永远能给他"没见过的事"

#### LLM 输入/输出约束框架（D 类核心设计问题）

**输入设计（LLM "看到"什么）**：
- 世界状态快照：当前场景/实体/资源数值（最小相关集，不是全局）
- 玩家行为历史：最近 N 次关键操作
- 角色档案：性格/目标/记忆摘要
- 规则约束集：你能做什么/不能做什么
- 当前事件：触发本次决策的事件

**输出约束（LLM "能改"什么）**：
- **白名单模式**：LLM 只能从枚举动作空间中选择并填参数
- **幅度限制**：每个数值改动有上下限（如心情 -20~+20），防止一次拉满
- **禁止列表**：硬编码不可触碰的行为（如直接杀死玩家、破坏核心规则）
- **程序化校验**：LLM 输出先过规则检查，不合法直接用降级方案

#### "斗蛐蛐"洞察：玩家的观赏性需求

来自黑神话悟空的"斗蛐蛐"现象揭示了一个核心玩家需求：

> **"我想看两个有个性的实体，在我设定的条件下，产生我无法预测的结果。"**

拆解为三层：
1. **个性化**：实体不是无名怪物，是"有身份的"——玩家有情感投射
2. **可设定**：我能选谁、什么规则、什么条件——玩家有控制感
3. **不可预测**：结果是"涌现"出来的——玩家有惊喜感

传统"斗蛐蛐"靠数值+动画。LLM 版的"斗蛐蛐"中，角色可以**真的有性格**——会怯场、会愤怒、会使诈、会有自己的战斗哲学。从"看数值碰撞"变成"看两个有灵魂的实体博弈"。

#### "伪开放世界"问题与 LLM 的解法

传统开放世界本质是"大量人力预设的有限内容集合"——玩家玩到一定程度，就知道"这个世界有尽头"。

LLM 可以带来的独到体验：
- **惊喜感**：NPC 做出设计师都没预设过的合理行为
- **出乎意料感**：世界对玩家行为产生连锁反应，而非预设脚本
- **无尽探索感**：系统性内容生成让世界永远有新组合

关键实现路径：**异步回合制**（参考鬼谷八荒的世界模型）
- 世界以"天/回合"为单位推进，每回合结束批量调用 LLM
- 不需要实时响应，延迟不是问题
- 状态可快照，容易序列化给 LLM
- 复杂度可控，一次只处理"这一天发生的事"

### D1. "小村庄领主"（Village Lord） `SELECTED`

> **已选定为立项方向。** 详细项目简报见 `AIvilization World/项目简报-小村庄领主.md`

你是一个小村庄的领主。村里有 6-8 个村民，各有独特性格、目标与记忆。每天你有丰富的传统游戏操作（管理资源/建造设施/探索周边/制作道具），同时可以与村民交流、颁布政策。AI 在后台驱动村民的自主行为和关键事件——你在游戏中感受到的是"这个世界会给我惊喜"，而不是"我在和 AI 聊天"。

- **基础层（不依赖 AI，即时反馈）**：资源采集/建造/制作/探索/日程管理——像星露谷一样，每天有充实的事可做
- **增强层（AI 有限介入）**：NPC 对话/关系事件/个性化请求——每 2-3 天出现一次 AI 驱动的独特互动
- **惊喜层（AI 关键时刻）**：突发危机/NPC 自发行为/结局编年史——罕见但令人难忘
- **设计原则遵循**：Gameplay First / 85-15 / 四层反馈 / 白名单介入 / 事件化输出 / 克制原则

#### 最小原型规模评估

| 维度 | 最小原型（1-2周） | 完整体验（1-3月） |
|------|-----------------|------------------|
| 世界规模 | 1 个村庄，6 个 NPC | 多区域，15-20 NPC |
| 传统玩法层 | 资源管理+简易建造+NPC日程 | 完整经营+探索+制作+季节 |
| AI 调用频率 | 每 2-3 回合 1-2 次 | 每回合 2-3 次 |
| 玩家交互 | 选择+短文本 | 自由文本+操作 |
| 世界推进 | 手动点击"下一天" | 半自动挂机+干预 |
| API 成本/天 | ~1-3 元 | ~10-30 元 |
| 涌现感 | 足够验证核心体验 | 支撑数十小时 |

## 更"震撼"的卖点包装（策划案亮点句）

- **玩家不是在选对话，而是在写规则**：输入一句话，世界就按这句话运行（受约束、可验收）。
- **每次通关都是一份"可执行契约/咒语/宪法"合集**：通关记录本身就是玩家创造的系统资产。
- **失败不是数值不够，而是"论证不成立/契约漏洞/咒语逻辑错误"**：失败也有学习与改写空间。
- **你的词汇就是你的弹药**：每个词只能用一次，创造力就是战斗力。
- **AI 不是你的工具，是你的对手、法官、或者整个社会**：LLM 不辅助你，LLM 就是游戏本身。
- **你看不见 AI，但你能感受到这个世界是活的**：每个村民有自己的想法，你只是领主。
- **没有预设结局，只有你和你的村庄写出的历史**：每一局都是独一无二的编年史。
- **95% 的时间你在玩游戏，5% 的时间游戏在给你惊喜**：AI 不抢戏，但你永远忘不了它登场的那一刻。 `NEW`

## 省时间的落地推进方式（8 小时友好）

- **只选 1 个核心循环**，最多 1 个辅助循环；先把"输入 → LLM → 解析/约束 → 状态变化 → 结算"写成闭环，再扩内容。
- **优先结构化输出**（JSON/标签/枚举），保证可解析、可测试、易做降级与兜底。
- **准备验收用例**：至少 5 条可复现用例，证明"LLM 结果可被玩家策略性影响"且"移除 LLM 会让核心循环不可用"。
- **先做不依赖 AI 的基础层**：确保关掉 AI 后游戏仍可操作、有基本乐趣，再叠加 AI 增强。 `NEW`
